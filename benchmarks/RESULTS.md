# 검증 결과 — 이름을 바꿔도 잡아내는가?

> 실행: `python benchmarks/poc_winnowing.py` (누구나 재현 가능)
> 측정일: 2026-07-12

## 검증 명제

> **"AI가 GPL 코드를 변수명만 바꿔서 재현했을 때, 우리는 잡아내는가?"**

이것이 Provenire의 존재 이유다.
GitHub Copilot의 `public code filter`는 **문자열이 거의 같을 때만** 걸러내므로,
**변수명만 바꾸면 그대로 통과한다.**

## 실험 설계

| 항목 | 내용 |
|---|---|
| **원본** | `qutebrowser/utils/utils.py :: elide_filename()` — 실제 **GPL-3.0-or-later** 코드 |
| **엔진 A `raw`** | 주석·공백만 제거 후 비교 (= Copilot 필터 수준) |
| **엔진 B `tokens`** | **식별자를 `ID`로 익명화** 후 비교 (= Provenire) |
| **지표** | containment = `|공유 지문| / |의심 코드 지문|` |
| **파라미터** | k=25 (k-gram), w=12 (winnowing 윈도우) |
| **임계값** | 30% 이상이면 "표절 의심" |

## 결과

| 변형 시나리오 | raw | **tokens** | 판정 |
|---|:--:|:--:|---|
| 1) 그대로 복사 | 100.0% | **100.0%** | 둘 다 탐지 |
| 2) 주석·독스트링 삭제 | 100.0% | **86.4%** | 둘 다 탐지 |
| 3) **변수·함수명 전부 변경** | **0.0%** | **100.0%** | ⭐ **tokens만 탐지** |
| 4) 이름변경 + 주석삭제 + 재포맷 | **0.0%** | **86.4%** | ⭐ **tokens만 탐지** |
| 5) 무관한 코드 (음성 대조군) | 0.0% | **0.0%** | 오탐 없음 |

### k-gram 길이별 민감도 (시나리오 3)

| k | raw | **tokens** | 음성 대조군 오탐 |
|:--:|:--:|:--:|:--:|
| 10 | 17.1% | **100.0%** | 0.0% |
| 15 | 8.3% | **100.0%** | 0.0% |
| 20 | 2.6% | **100.0%** | 0.0% |
| 25 | 0.0% | **100.0%** | 0.0% |
| 30 | 0.0% | **100.0%** | 0.0% |

**k를 어떻게 잡아도 `tokens`는 100% 탐지 / 0% 오탐.**
반면 `raw`는 k가 커질수록 **0%로 붕괴**한다.

## 결론

> **변수명을 바꾸면 raw 방식(= Copilot 필터 수준)은 완전히 실패한다(0%).
> 토큰 정규화 방식은 100% 탐지하면서 오탐은 0%다.**

식별자 익명화가 **핵심 해자(moat)** 임이 코드로 입증되었다.

## 한계 (정직하게)

이 실험은 **Go/No-Go 판단용 PoC**다. 다음이 남아 있다.

1. **단일 함수·단일 언어(Python)** — 코퍼스 확대 필요
2. **변형을 스크립트로 생성** — 실제 LLM에게 "다시 써줘"로 재생성시킨 테스트가 더 현실적
   (LLM은 로직 순서·사용 API까지 바꿀 수 있다)
3. **대규모 인덱스에서의 오탐 미측정** — 보일러플레이트(getter/setter, 표준 관용구)가
   서로 매칭되는 문제는 별도 검증이 필요하다. → 관용구 필터 + LLM 2차 판정으로 대응 예정
